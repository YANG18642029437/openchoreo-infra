# K3s etcd SSD Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为三台 K3s server VM 分别添加一块位于 PVE `SSD1` 的 20 GiB 虚拟磁盘，并以逐节点、可停止、可回滚的方式把 embedded etcd 数据迁移到 SSD。

**Architecture:** Terraform 只声明 VM 120–122 的独立 `scsi1`，并通过 fail-closed 的 JSON plan 门禁阻止替换、删除或无关漂移。Ansible 使用不导入 `site.yml` 的专用 role 和滚动 playbook，先审计唯一新盘，再在显式格式化开关下创建 ext4、复制 etcd、建立 bind mount，并在每台节点恢复健康后继续。

**Tech Stack:** Terraform 1.15.8、`bpg/proxmox` 0.111.0、Ansible、`community.general`、`ansible.posix`、K3s embedded etcd、Bash 3.2、Ruby、Python 3。

---

- [ ] **Step 3: 创建保存 plan 的 fail-closed 验证器**

创建可执行文件 `scripts/verify/terraform-etcd-ssd-plan.sh`：

```bash
#!/usr/bin/env bash
set -euo pipefail

if [ "$#" -ne 1 ]; then
  printf 'usage: %s <terraform-plan-file>\n' "$0" >&2
  exit 2
fi

repo_root="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
tf_dir="$repo_root/terraform/environments/homelab"
plan_file="$1"

test -f "$plan_file" || {
  printf 'missing Terraform plan: %s\n' "$plan_file" >&2
  exit 1
}
command -v terraform >/dev/null 2>&1 || {
  printf 'missing command: terraform\n' >&2
  exit 1
}
command -v python3 >/dev/null 2>&1 || {
  printf 'missing command: python3\n' >&2
  exit 1
}

terraform -chdir="$tf_dir" show -json "$plan_file" |
  python3 -c '
import json, sys

payload = json.load(sys.stdin)
expected = {
    "module.k3s_nodes[\"ocp-node-01\"].proxmox_virtual_environment_vm.this",
    "module.k3s_nodes[\"ocp-node-02\"].proxmox_virtual_environment_vm.this",
    "module.k3s_nodes[\"ocp-node-03\"].proxmox_virtual_environment_vm.this",
}
changes = payload.get("resource_changes")
if not isinstance(changes, list):
    raise SystemExit("plan has no resource_changes array")

changed = []
for item in changes:
    actions = item.get("change", {}).get("actions")
    if actions == ["no-op"]:
        continue
    address = item.get("address")
    if address not in expected or actions != ["update"]:
        raise SystemExit(f"unexpected Terraform change: {address} actions={actions}")
    before = item["change"].get("before") or {}
    after = item["change"].get("after") or {}
    before_disks = before.get("disk") or []
    after_disks = after.get("disk") or []
    before_by_interface = {disk.get("interface"): disk for disk in before_disks}
    after_by_interface = {disk.get("interface"): disk for disk in after_disks}
    if None in before_by_interface or None in after_by_interface:
        raise SystemExit(f"{address} has a disk without an interface")
    if "scsi1" in before_by_interface:
        raise SystemExit(f"{address} already has scsi1 before this plan")
    disk = after_by_interface.get("scsi1")
    if disk is None:
        raise SystemExit(f"{address} does not add scsi1")
    preserved_before = before_by_interface
    preserved_after = {
        interface: value
        for interface, value in after_by_interface.items()
        if interface != "scsi1"
    }
    if preserved_after != preserved_before:
        raise SystemExit(f"{address} changes an existing disk")
    expected_disk = {
        "datastore_id": "SSD1",
        "interface": "scsi1",
        "size": 20,
        "iothread": True,
        "discard": "on",
        "ssd": True,
    }
    for key, value in expected_disk.items():
        if disk.get(key) != value:
            raise SystemExit(f"{address} invalid {key}: {disk.get(key)!r}")
    changed.append(address)

if set(changed) != expected:
    raise SystemExit(f"changed VM set mismatch: {sorted(changed)}")
print("Terraform etcd SSD plan: PASS")
'
```

```bash
#!/usr/bin/env bash
set -euo pipefail

repo_root="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
defaults="$repo_root/ansible/roles/k3s_etcd_ssd/defaults/main.yml"
tasks="$repo_root/ansible/roles/k3s_etcd_ssd/tasks/main.yml"
playbook="$repo_root/ansible/playbooks/36-k3s-etcd-ssd.yml"
site="$repo_root/ansible/playbooks/site.yml"

fail() {
  printf 'FAIL: %s\n' "$*" >&2
  exit 1
}

for path in "$defaults" "$tasks" "$playbook" "$site"; do
  test -f "$path" || fail "missing ${path#$repo_root/}"
done

require_literal() {
  grep -Fq -- "$2" "$1" || fail "${1#$repo_root/} missing: $2"
}

require_literal "$playbook" 'hosts: k3s_servers'
require_literal "$playbook" 'serial: 1'
require_literal "$playbook" 'any_errors_fatal: true'
require_literal "$playbook" 'role: k3s_etcd_ssd'
require_literal "$playbook" 'ansible_play_hosts_all | length == 1'
require_literal "$defaults" 'k3s_etcd_ssd_allow_format: false'
require_literal "$defaults" 'k3s_etcd_ssd_mount_path: /var/lib/rancher/k3s-ssd'
require_literal "$defaults" 'k3s_etcd_ssd_bind_path: /var/lib/rancher/k3s/server/db'

for contract in 'lsblk' 'findmnt' 'k3s etcd-snapshot save' \
  'community.general.filesystem' 'ansible.posix.mount' 'rsync' \
  'systemd_service' 'bind,x-systemd.requires-mounts-for=' \
  'kubectl' '--for=condition=Ready' '/readyz?verbose'; do
  require_literal "$tasks" "$contract"
done

if grep -Fq '36-k3s-etcd-ssd.yml' "$site"; then
  fail 'destructive migration playbook must not be imported by site.yml'
fi

require_literal "$tasks" 'k3s_etcd_ssd_candidates | length == 1'
require_literal "$tasks" 'k3s_etcd_ssd_allow_format | bool'
require_literal "$tasks" 'inventory_hostname == ansible_play_hosts_all[0]'

printf 'PASS: K3s etcd SSD migration contract\n'
```

最后一个 `inventory_hostname` 契约用于确保每次 play 调用只在其首个受限节点停机前创建一次集群级快照。runbook 会按 `--limit` 单节点执行，因此三个节点各自迁移前都会获得一个新的恢复点。

- [ ] **Step 2: 实现磁盘审计与显式格式化停止点**

在 `ansible/roles/k3s_etcd_ssd/tasks/main.yml` 中先实现：

```yaml
---
- name: Read the root backing device
  ansible.builtin.command:
    argv: [findmnt, -rn, -o, SOURCE, /]
  register: k3s_etcd_ssd_root_source
  changed_when: false

- name: Read block device inventory as JSON
  ansible.builtin.command:
    argv:
      - lsblk
      - --bytes
      - --json
      - --output
      - NAME,PATH,TYPE,SIZE,FSTYPE,MOUNTPOINTS
  register: k3s_etcd_ssd_lsblk
  changed_when: false

- name: Select the unique 19-21 GiB data disk
  ansible.builtin.set_fact:
    k3s_etcd_ssd_candidates: >-
      {{
        (k3s_etcd_ssd_lsblk.stdout | from_json).blockdevices
        | selectattr('type', 'equalto', 'disk')
        | selectattr('size', 'ge', k3s_etcd_ssd_minimum_disk_bytes)
        | selectattr('size', 'le', k3s_etcd_ssd_maximum_disk_bytes)
        | list
      }}

- name: Require exactly one non-root SSD candidate
  ansible.builtin.assert:
    that:
      - k3s_etcd_ssd_candidates | length == 1
      - k3s_etcd_ssd_candidates[0].path not in k3s_etcd_ssd_root_source.stdout
      - k3s_etcd_ssd_candidates[0].fstype in [none, '', k3s_etcd_ssd_filesystem]
      - >-
        k3s_etcd_ssd_candidates[0].mountpoints in [[none], []]
        or k3s_etcd_ssd_mount_path in k3s_etcd_ssd_candidates[0].mountpoints
    fail_msg: >-
      Refusing disk changes; expected one managed 19-21 GiB non-root disk,
      candidates={{ k3s_etcd_ssd_candidates }},
      root={{ k3s_etcd_ssd_root_source.stdout }}.

- name: Detect an already migrated node
  ansible.builtin.set_fact:
    k3s_etcd_ssd_already_migrated: >-
      {{
        k3s_etcd_ssd_candidates[0].fstype == k3s_etcd_ssd_filesystem
        and k3s_etcd_ssd_mount_path in k3s_etcd_ssd_candidates[0].mountpoints
      }}

- name: Reject a partially configured existing filesystem
  ansible.builtin.assert:
    that:
      - >-
        k3s_etcd_ssd_candidates[0].fstype in [none, '']
        or k3s_etcd_ssd_already_migrated
    fail_msg: >-
      Refusing to adopt an existing filesystem that is not already mounted at
      {{ k3s_etcd_ssd_mount_path }}.

- name: Require explicit authorization before first format
  ansible.builtin.assert:
    that:
      - k3s_etcd_ssd_allow_format | bool
    fail_msg: >-
      Disk audit passed. Re-run with -e k3s_etcd_ssd_allow_format=true
      only after the execution-time confirmation.
  when: not k3s_etcd_ssd_already_migrated
```

不要在候选盘已有文件系统时自动采用它。只有 ext4 已挂载在准确目标路径时才视为已迁移，并跳过 snapshot、停服、格式化和复制；最后仍执行 bind mount 与健康验证。首次迁移命令按 `--limit` 单节点执行并记录结果。

- [ ] **Step 4: 实现停止、格式化、复制校验和 bind mount**

继续加入以下核心任务；其中 `community.general.filesystem` 前必须重复候选盘断言：

```yaml
- name: Install the required rsync package
  ansible.builtin.apt:
    name: rsync
    state: present
    update_cache: true
    cache_valid_time: 3600
  when: not k3s_etcd_ssd_already_migrated

- name: Stop K3s on the current node
  ansible.builtin.systemd_service:
    name: k3s
    state: stopped
  when: not k3s_etcd_ssd_already_migrated

- name: Re-read the target disk immediately before formatting
  ansible.builtin.command:
    argv:
      - lsblk
      - --bytes
      - --json
      - --output
      - NAME,PATH,TYPE,SIZE,FSTYPE,MOUNTPOINTS
      - "{{ k3s_etcd_ssd_candidates[0].path }}"
  register: k3s_etcd_ssd_format_recheck
  changed_when: false
  when: not k3s_etcd_ssd_already_migrated

- name: Freeze the rechecked target disk
  ansible.builtin.set_fact:
    k3s_etcd_ssd_format_target: >-
      {{
        (k3s_etcd_ssd_format_recheck.stdout | from_json).blockdevices
        | selectattr('path', 'equalto', k3s_etcd_ssd_candidates[0].path)
        | list
      }}
  when: not k3s_etcd_ssd_already_migrated

- name: Require the target disk to remain unused before formatting
  ansible.builtin.assert:
    that:
      - k3s_etcd_ssd_format_target | length == 1
      - k3s_etcd_ssd_format_target[0].type == 'disk'
      - k3s_etcd_ssd_format_target[0].size >= k3s_etcd_ssd_minimum_disk_bytes
      - k3s_etcd_ssd_format_target[0].size <= k3s_etcd_ssd_maximum_disk_bytes
      - k3s_etcd_ssd_format_target[0].fstype in [none, '']
      - k3s_etcd_ssd_format_target[0].mountpoints in [[none], []]
      - k3s_etcd_ssd_format_target[0].path not in k3s_etcd_ssd_root_source.stdout
  when: not k3s_etcd_ssd_already_migrated

- name: Create the SSD ext4 filesystem
  community.general.filesystem:
    dev: "{{ k3s_etcd_ssd_candidates[0].path }}"
    fstype: "{{ k3s_etcd_ssd_filesystem }}"
  when:
    - not k3s_etcd_ssd_already_migrated
    - k3s_etcd_ssd_allow_format | bool

- name: Read the SSD filesystem UUID
  ansible.builtin.command:
    argv: [blkid, -s, UUID, -o, value, "{{ k3s_etcd_ssd_candidates[0].path }}"]
  register: k3s_etcd_ssd_uuid
  changed_when: false

- name: Mount the SSD by UUID
  ansible.posix.mount:
    path: "{{ k3s_etcd_ssd_mount_path }}"
    src: "UUID={{ k3s_etcd_ssd_uuid.stdout }}"
    fstype: "{{ k3s_etcd_ssd_filesystem }}"
    opts: defaults,noatime,errors=remount-ro
    state: mounted

- name: Create the SSD etcd data directory
  ansible.builtin.file:
    path: "{{ k3s_etcd_ssd_data_path }}"
    state: directory
    owner: root
    group: root
    mode: "0700"

- name: Copy etcd data to the SSD
  ansible.builtin.command:
    argv:
      - rsync
      - -aHAX
      - --numeric-ids
      - --delete
      - "{{ k3s_etcd_ssd_bind_path }}/"
      - "{{ k3s_etcd_ssd_data_path }}/"
  register: k3s_etcd_ssd_rsync
  changed_when: true
  when: not k3s_etcd_ssd_already_migrated

- name: Read source etcd copy statistics
  ansible.builtin.shell: >-
    set -o pipefail && find {{ k3s_etcd_ssd_bind_path | quote }} -xdev -type f
    -printf '%s\n' | awk '{ bytes += $1; count += 1 } END { print count, bytes }'
  args:
    executable: /bin/bash
  register: k3s_etcd_ssd_source_stats
  changed_when: false
  when: not k3s_etcd_ssd_already_migrated

- name: Read SSD etcd copy statistics
  ansible.builtin.shell: >-
    set -o pipefail && find {{ k3s_etcd_ssd_data_path | quote }} -xdev -type f
    -printf '%s\n' | awk '{ bytes += $1; count += 1 } END { print count, bytes }'
  args:
    executable: /bin/bash
  register: k3s_etcd_ssd_target_stats
  changed_when: false
  when: not k3s_etcd_ssd_already_migrated

- name: Require matching source and SSD statistics
  ansible.builtin.assert:
    that:
      - k3s_etcd_ssd_source_stats.stdout == k3s_etcd_ssd_target_stats.stdout
  when: not k3s_etcd_ssd_already_migrated

- name: Bind mount the SSD etcd directory
  ansible.posix.mount:
    path: "{{ k3s_etcd_ssd_bind_path }}"
    src: "{{ k3s_etcd_ssd_data_path }}"
    fstype: none
    opts: "bind,x-systemd.requires-mounts-for={{ k3s_etcd_ssd_mount_path }}"
    state: mounted
```
