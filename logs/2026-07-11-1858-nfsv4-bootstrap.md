# VM 130 NFSv4 初始化

## 审批与磁盘安全检查

- 用户在展示设备、容量、文件系统和挂载状态后确认继续格式化。
- Ansible 只识别到一块 390-410 GiB 数据盘：`/dev/sdb`，实际容量 429496729600 bytes。
- 首次操作前 `/dev/sdb` 没有文件系统且未挂载；系统盘 `/dev/sda` 不在候选集合中。
- Playbook 默认 `nfs_allow_format=false`，首次格式化必须显式传入授权变量。

## 执行结果

- `/dev/sdb` 已创建 XFS，并通过 UUID 挂载到 `/srv/openchoreo`。
- 已创建 Harbor、共享构建产物及备份目录。
- NFSv4 根导出限制为 `192.168.2.0/24`，启用 `root_squash`。
- `nfs-server` 状态为 active。
- 从 VM 120 挂载 `192.168.2.183:/` 后，`shared` 目录写入和删除测试通过。
- 未记录文件系统 UUID、SSH 私钥或其他敏感内容。

## 故障与恢复

首次配置在写 `/etc/exports.d/openchoreo.exports` 时发现目标目录不存在。此时 XFS 和挂载已经完成，但尚未导出。补充显式目录创建任务后幂等续跑成功，没有重新格式化磁盘。

## 最终验证

- `scripts/verify/nfs.sh` 返回 0。
- 不携带格式化授权变量再次运行 NFS Playbook，结果为 `changed=0`。
- 磁盘安全断言允许的稳定状态仅为未挂载空盘，或 XFS 挂载到 `/srv/openchoreo`。

下一步是生成私有 K3s server token，并在 VM 120 上执行 embedded-etcd `cluster-init`，随后按顺序加入 VM 121 和 VM 122。开始集群初始化前需要再次确认。
