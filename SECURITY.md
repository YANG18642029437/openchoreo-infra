# 安全边界

## 本地敏感信息

所有静态落盘的凭据、私钥、kubeconfig、令牌、Terraform state、PKI 材料和备份只能存放在仓库根目录的 `.private/` 中。`.private/` 及其子目录权限必须为 `0700`，其中新建文件的默认权限必须为 `0600`。这些内容不得提交到 Git。

运行时可以读取 `.private/` 中受保护的文件，再通过环境变量或标准输入注入敏感值；环境变量和标准输入不是静态存储位置，但其源文件仍必须满足上述目录和权限要求。密码和令牌不得通过命令行参数传递，避免它们出现在 shell 历史、进程列表或操作日志中，并确保脚本不会打印敏感值。

## 自动化工具边界

Terraform 只能从 `PROXMOX_VE_*` 环境变量读取 Proxmox 凭据，不得把凭据写入 Terraform 配置、变量文件、state 或命令行参数。

Ansible 必须使用 SSH 密钥认证。凡是可能读取、处理或返回敏感信息的任务必须启用 `no_log: true`，避免凭据进入终端输出和日志。

## 误提交响应

发现敏感信息被意外提交后，必须按以下顺序处理：

1. 立即停止推送，防止泄露范围扩大。
2. 轮换受影响的凭据，并撤销旧凭据。
3. 清理 Git 历史中的敏感内容。
4. 完成凭据轮换和历史清理后，运行 `REQUIRE_GITLEAKS=1 ./scripts/verify/secrets.sh`，强制同时完成正则扫描和 gitleaks 扫描；gitleaks 不可用时不得跳过。
5. 记录安全事件、影响范围、处置过程和后续改进。
